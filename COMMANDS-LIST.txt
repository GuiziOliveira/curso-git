git init - Inicia a "tracking" em um repositório.
ls - lista de arquivos do diretório.
git status - mostra a situação atual do diretório.
git ls-files - mostra os arquivos já "trakeados" pelo git.
git add - adiciona arquivo ao repositório.
git whatchanged - Verifica o arquivo que foi mudado em cada versão do respositório.
git whatcanged -p  - Mostra o que foi alterado no repositório (linhas adicionadas ou excluidas).
git pull -u - Da o comando pull trackeando o repositório local ao repositório remoto.
git branch -t origin/repositirio - Cria o repositorio local já trackeado com o remoto.
gir branch -b - Cria o repositorio local e dá checkout para ele.
git branch -d - Deleta o branch
git branch -r - Lista somente os branchs remotos.
git branch -a - Lista todos os branchs local/remote.
git checkout -t origin/repositorio - Cria branch local, mudança para essa nova branch criada, linka branch local com remoto.
git fetch origin - Este comando verifica todas as atualizações que foram realizadas no repositório de atalho origin.
git rebase repositorio-refernecia - Atualiza o repositório atual com o de referênncia (Caso haja commits feitos anteriormente que não foram puxados para o repositório).
git merge repositório - Faz merge no repositório atual com o repositório selecionado.
git reset - Com este comando, as alterações são removidas do histórico local de commits, do index, mas não doworking directory`.
git reset --soft - Com este comando, as alterações são removidas do histórico local de commits, mas não são removidas do index.
git reset --hard - Com este comando, as alterações são removidas do histórico local de commits e também tanto do index quanto do working directory, permanentemente.
git reset HEAD - retorna alterações do commit anterior (se as alterações não foram comitadas ainda).
git reset código do commit - retorna as alterações para um commit especifico (mesmo já estando commitado) levando para estado de modificação.
git revert código do commit - cria commit para desfazer o commit selecionado.
git stash - Guarda alterações que ainda não foram commitadas.
git stash list - Verifica o que está em stash.
git stash pop - Retorna alterações do ultimo stash salvo.
git stash apply código do stash - Retorna stash que foi selecionado.
git stash drop - Remove alteração temporária.
git bisect start - Inicia modo bisect (busca de commit especifico).
git bisect bad código do commit - Indica qual commit não está bom, para que ele filtre antes disso.
git bisect good código do commit - Indica qual commit está bom, para que ele filtre a partir dele.
git fetch origin - Este comando verifica todas as atualizações que foram realizadas no repositório de atalho origin.
